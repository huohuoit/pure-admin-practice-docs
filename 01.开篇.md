# 写在最前面

有朋友们会问，你现在的上班通勤时间从走路几分钟到坐公交半小时，要早起更久回家更晚，会不会有些难受。

距离搬家已经过去半年，实际上我很享受这个多出来的半小时。不刷视频，不看小说，戴上耳机，打开高德地图语音提醒，播放适合当时心情的音乐。闭上眼睛，映入脑海的可能是昨天的经历，今天的计划，未来的走向，可能会想昨天的某个问题，某一个技术，也或者是某一个人某一件事。每一次短短的 30 分钟，都可能是一次小小的升华。是的，这个过程我们也可以把它叫做冥想。而关于开源项目 [pure-admin](https://github.com/pure-admin) 及其生态的思想与源码解析的想法，就此迸发。

就这样在每个工作日的公交上，来来回回构思了无数次，要怎么去写这个系列。最后决定将文章的脉络分为两条线，交叉进行。

## 路线一：项目实践与解析

最早接触到 pure-admin 大约是在 2021 年 8 月，而同时在 9 月，`Vue3.0`正式发布为`Vue3.2`，也就是后来默认的 3.0 版本。[vue-pure-admin](https://github.com/pure-admin/vue-pure-admin) 也在后续迅速的换上了`Composition-API`，在当时 Vue3 兴起的风口下，vue-pure-admin 与我使用的技术栈 `Vue3 + Element-Plus + Vite + TS` 完美的契合。期间 pure 持续保持着依赖版本的最新性与稳定性，我也几乎关注并保持更新了作者的每一个 commit，以至于后来作者在群里说我是研究他源码最深的人。一个在前端界家常便饭的话题，admin 后台管理系统，pure 作为个人开源项目 在 github 上从最早的小几百 star 到现在将近 8K 的 star，不得不说是非常成功的。当然，那时候我也针对某些细节对比了多个 star 更高的 admin 项目，比如后台管理系统的祖师爷 [vue-element-admin](https://github.com/PanJiaChen/vue-element-admin)、[vue-vben-admin](https://github.com/anncwb/vue-vben-admin)，不管从技术新颖的程度上，还是初代 Vue3 开发者的代码编写的方式上来看，个人觉得`vue-pure-admin`对我来说是最合适的选择。

现在 pure 已经趋于稳定，也有了保姆级的文档 [pure-admin-doc](https://yiming_chang.gitee.io/pure-admin-doc/)，里面详细介绍了项目的使用以及一些作者记录的开发问题。很庆幸，曾经作为 pure 团队的一员，参与了 `pure-admin-doc` 的搭建与初版编写（后来因为条件和工作环境等原因无法继续编写）。但是对于 `vue-pure-admin` 的使用者来说，尤其是初级前端开发工程师和相关技术栈初次使用的开发者来说，依然存在很多的问题（尤其是怎么更改或者增加项目中某个模块的功能），更详细的文档与实践依然迫切需要。2022 年上半年，我写过一个系列的关于 pure 的实践文章（其实没写完），但是后来 pure 进行了大范围的重构，也就不适用了。恰逢在公司目前一个后台管理系统的发布之际，想趁这个时间点，重新梳理一下这个项目从 0-1 的实践，以及开发过程中，我们所遇到的问题做的选择，还有未来可能需要做的优化。比如我们怎么选择合适的插件依赖进行搭配，需不需要什么替代方案来渡过当前研发阶段。比如 Vue3 中，组件的开发，是使用 `defineComponent`、`setup`、`render` 函数还是 `tsx`。他们之间的通信，是选择 `pinia`、`defineProps/defineEmits`、`expose/ref`、`useAttrs`、`v-model`、`provide` / `inject` 还是 `mitt` 。又比如 `CSS` 处理器或框架的选择，`less` 还是 `sass`，`TailWind CSS`、`WindCSS` 还是 `Unocss`。

虽然说后台管理系统对于每个前端人都耳熟能详，也或许每个前端人都做过一个后台管理系统。但是想真正做好一个后台管理系统，其实并不简单。在 pure 的近千次 `commit` 中，在自己的项目落地实践中，发生的种种碰撞与摩擦，都值得去做一些反思与总结。将整个系统细分下来，都是前端这条路上的一个个缩影，每个缩影中，也都有着一个个前人走过的坑，它们可能不仅仅存在于后台，也可能是其他 PC 端软件、APP 亦或者小程序。不管是为了自己当前项目的总结复盘或技术的查漏补缺，还是围绕 pure 这个开源项目做详尽的开发文档，方便后来的优化与维护，都少不了这份文档的编写。

## 路线二：前端三年之约

时间非常的巧妙，到这个时间节点，我真正接触前端也将满三年整，再过四个月也将正式工作满三年。在最初步入前端行业的时候，我给自己定了 1-3-5 年的时间节点规划（具体我就不献丑咯），经历不华丽，但是很幸运。

**毕业后，在一年结尾之时顺利地完成了跳跃，庆幸又觉得应该**。第一份工作进入了一家大家口中不太好的公司，幸运的是，我进入了一个对我来说绝佳的项目组。业务线的新形成，项目组的刚成立，团队人员的初始招聘，技术框架的探索实践，很好，就是我这个应届生最最需要的。很幸运，由于项目组刚成立，并没有多少工作量，给了我充足的时间去学习，也给了我更多地机会去犯错。在导师的带领下，经历了，从一系列移动端项目完成的的 0-1 过程；从客户到产品、项目经理，到硬件工程师、通信工程师，到 UI 设计，再到我们前端开发，最后通过严格的测试；从框架的搭建，到各种小型库的开发或使用，到规范的形成，到团队模板的推出（在这里我第一次完整的参与了一个项目到发起到最后落地的整套流程）。那个时候最骄傲的事情就是，同事们（近 20 个前端），都能使用我做的框架模板，使用我写的组件和库，能跟各个部门熟稔和谐地协调沟通，甚至能指导一些三五年经验的同事代码。虽然经常一个人在公司待到凌晨（项目组基本五点半下班就都走了，我是自己留下来学习哈！），但是还是很开心，因为我进来的第一天就准备好了一年后跑路。一切都是计划好的，刚毕业不久的我，需要一个更好的平台，需要一个能对我成长更有利的地方，也因为...（DDDD）。为此，努力，乐此不疲，因此，鱼跃龙门（找不到词了，夸张一下下）。

**将满三年之际，我仍在追逐自身的飞跃，坚定又相信自己**。虽然比不上大厂那些核心优异的部门，虽然比不了抖音淘宝这种流量巨大的平台，但我还是非常幸运的在合适的时间选择到了适合自己的地方。是的，这时候 `Vue3` 及其周边生态正处于风口，刚刚熟悉完 `Vue2` 的我也正准备扑上浪潮，这里，给了我机会，给了我环境。同样是新公司新团队的成立，新技术的自由探索，公司层面上也算是一次改革。最大的不同是，氛围，我们年轻，有活力，追求创新，追求愉悦的工作模式，开放式的分享式的工作环境也让我接触到了产品设计、项目管理、团队管理、交流协作、算法及后端等多方面的知识与理念。也是在这里，我开始了 `Vue3` 的学习，以及它的当前最流行的一整套技术栈，然后开始了第一个艰难的项目，重构。没错，一年工作经验的我，想都不敢想，重构一个业务繁多的项目。当时是我负责用从没用过的 `Vue3 + Vite + TS` 对一个具有一些较重历史包袱的后台管理系统（Vue2）进行重构，在 pure 和团队 leader，以及一些网友的帮助下，还是顺利的完成啦！虽然现在看来并不太难，但是对于那时候的我来说，极具挑战性。再到后来其他项目的开发，团队的扩展，协作与规范，各种问题的发现与解决（后续文章会慢慢说到），我仍然在迅速成长。到这里，还是想说，我是幸运的，公司与团队的发展与方向，与我自身的路线是非常一致的，路线越来越清晰的同时，我也能跑得越来越快。相信自己，相信团队，一起飞跃吧 ~

只有短短不到三年的工作经验，其实也只能说是一只前端界的小小菜鸡（确实还是很菜，跟很多同龄的大佬们比差距太明显了~），所以更想在这个特殊的节点，对自己三年的一些沉淀做一轮总结。毕竟，你可以不断地更替你的技术，你身边可以走过一个又一个熟悉的人，但你一辈子，这样的三年只有一次。你需要在这个时候，发现一些过去的什么，预知以后的一些什么，这些不仅仅是技术，也不仅仅是什么人什么事，可能就只是你“自己”。请努力，攒起对未来的，希望。

那么，就从写下这个系列文章开始。

到这里，我准备改变像曾经一样机械性写技术文章的想法，转向纪录片式口吻的方式进行这次实践与文档的编写，以我即你，你即我的角度去融入，去完成创作，以及对自己的一次洗礼。

有逻辑性地，想到什么写什么。这才是，我想要的技术，我想要的自己。

## 一些话

> 如果你刚入行前端，想要有一个详细可靠的从 0-1 的项目实践。

说实话现在的技术环境确实太缺乏这样的实践与分享了，一方面这样的分享太耗费时间与精力了，对于每个人都可以说是得不偿失，一方面类似的优异资源都需要付费（购买或者培训机构）。但并不是所有人都承担的起学费，并不是所有人都愿意去踩坑，也并不是所有人都能真正有所得。说这些其实是想到了自己大四那会的亲身经历，有很多人跟我一样是转行过来的，没有基础没有方向甚至可以说是没有对这行的认知，只知道计算机出路好。虽然当时我也是抱着对计算机的热爱走进这一行，但是面对高额的培训机构学费，面对未知又复杂的计算机环境，还是望而却步，选择自己在家苦苦自学。最后在考研失败、疫情爆发、基础薄弱的情况下，拖着疲惫的身躯进入艰难的就业市场，在已经毕业，七月底才找到第一份不太好的工作。所以，我想做一次这样的分享，因为我自己，曾经就是某某某，我，就是你们。

> 如果你也像我一样想对自己 3 年左右的工作进行一次技术和思想上的总结

3 年说长不长，说短也不短，若要用一句话来描述这三年的变化，我觉得这样能非常贴切的概括起来：当你准备开发一个模块的时候，从“我要怎么做”，到“我能不能这样做”，变成“我应该这样做”。比如 pure-admin，在拿到它的时候，我们通常会想，怎么将它使用到我们的实际项目中去。在使用的过程中，我能不能更改框架既定的某些逻辑，而不产生 Bug，保证项目不同环境的正常运行。同时在进行开发的时候，对框架某些地方提出质疑并给出优化方案（实际上在 pure 基本开发稳定前大家经常会在群里提出问题，当然你也可以提 issues 或者 pr）。这其实是一个比较常见也比较简单的过程，但我们不得不承认这是个人背后缓慢沉淀的结果。当然这也并不意味着你有了三年经验，会了这些，就是一名优秀的前端开发。真正优秀的人毕竟是少数，我们没有优异的学历，没有顶尖的天赋，但没有什么能阻止我们慢慢地努力，慢慢地沉淀。在接下来对项目的各个模块进行实践的时候，我也会基本遵循这个过程，去解决各种各样的问题（实际开发时，我们确实遇到了很多问题）。所以在这个系列里，跟着我一点一点地回忆，一起慢慢地记录总结。

> 如果你对自己未来的发展有一些疑虑（包括我自己）

第一年，你打好了基础，完成了跳跃，第三年，你有了自己的体系与思想，完成了飞跃，还有第五年在等着你，不是吗。关于未来发展这个问题，以及最近非常火的“前端已死”，掘金、知乎上已经有很多非常具有参考价值的文章，各方面不足的我其实是没有多少资格去讨论的，想说的就是，没有必要的焦虑，有必要的做好自己。

不过有几点，或许可以作为参考，其实也都是我自己在 3-5 年期间需要考量与定夺的事情（未来总是掌握在自己手中的）：

- **增强自己的不可替代性（技术方向上）**：你可以选择这些业务让自己处于团队中更稳定的一把好手——企业级中后台`node`业务（积累重成本高）、企业级`sass`服务（复杂）、基于`web`的重交互场景与业务（`WebGL`、`WebGPU`、地图、游戏、数据可视化等）、基于`Web`的工业化软件（`Web`版`excel`、`photoshop`等）。
- **路线转型**：往更深处走基本就是技术专家、架构、管理这三大类了（这也是我在未来这 3-5 年所要考虑的核心问题）。当然，具体还要看公司的发展看你的机遇，自身方向能够与公司的发展贴合那是再好不过。
- **工程师软素质提升**：这点我们可以从简历来看，近年来不管是对招聘者还是求职者来说，简历基本上千篇一律，都是那些东西。要想在如今艰难的就业环境中脱颖而出，解决这种同质化问题迫在眉睫。这里其实学历有着挺大的影响，而学历某种程度上代表的不外乎就是你的学习能力、理解与总结能力、逻辑思维强度。反过来看，学历只是一种表现，所以你可以选择换一种方式去体现，比如写文章（**文档对团队的帮助是不可估量的**），比如做技术视频，比如考证。也别忘了，沟通表达能力、规划能力也是帮助你吸引面试官眼球的一把好手。
- **升学**：其实学历对上面提到的管理岗就有不可或缺的影响。毕业前没有考到自己想上的研究生其实是很遗憾的，虽然现实很难再去考一次，但如果能再有一次机会，我一定不会放过。
- **考公**：对某些人来说，是一个非常不错的选择。但是面对现在竞争的越来越激烈，以及国内某些形式的不容乐观，需谨慎。（我的话大概率是不会的，躺平不适合我这更倾向于坻坜前行的性格）
- **多读书**：细心点你会发现，身边那些成功的或者生活的不错的人，其实大多有一个共同点，就是愿意去读书。读书不是应试，而是一辈子的事情，越到后面，你越会发现，人就应该多读书。
- **热爱生活，身体健康**：没有什么比你自己的身体与生活更重要，作为程序员的我们，请保证好自己的身体健康，请热爱属于你自己的生活！勇敢一点，多去看看世界，毕竟你永远不会再像现在一样年轻了。

就先说到这，接下来我们慢慢聊，希望这个系列的文章可能会给你带来一些帮助或者参考。另外，本人毕竟技术与经验有限，所写所述难免会有一些错误，请各位大佬们多多指教，一起进步！

## 开发准备

注： 本项目主要基于[vue-pure-admin](https://github.com/pure-admin/vue-pure-admin)源码，并参考其他优秀相关开源项目做一些优化，进行 0-1 的后台管理系统开发。

### 主要开发技术栈

- 核心前端框架：[Vue3.js](https://cn.vuejs.org/)（及其官方生态）
- UI 组件库：[Element-Plus](https://element-plus.gitee.io/zh-CN/)
- 项目构建工具：[Vite](https://cn.vitejs.dev/)
- 状态管理库： [Pinia](https://pinia.vuejs.org/zh/)
- Javascript 库：[Typescript](https://www.typescriptlang.org/zh/)

### 开发工具推荐

- 编辑器：[VSCode](https://code.visualstudio.com/)，大神才用的编辑器，用了你就是大神

- 聪明的 ChatGPT：懂的都懂，2023 年开发与学习的必备神器！记得注意一些提问技巧，抓住机会好好利用这个工具去超越别人吧！
- AI Code：[Cursor](https://www.cursor.so/)（免费免费免费！）。帮你写代码，帮你写注释，帮你解析源码，相信我，一定要用它！（当然[copilot](https://github.com/features/copilot)也必须争一争）
