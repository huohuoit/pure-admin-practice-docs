# 写在最前面

有朋友们会问，你现在的上班通勤时间从走路几分钟到坐公交半小时，要早起更久回家更晚，会不会有些难受。

距离搬家已经过去半年，实际上我很享受这个多出来的半小时。不刷视频，不看小说，戴上耳机，打开高德地图语音提醒，播放适合当时心情的音乐。闭上眼睛，映入脑海的可能是昨天的经历，今天的计划，未来的走向，可能会想昨天的某个问题，某一个技术，也或者是某一个人某一件事。每一次短短的 30 分钟，都可能是一次小小的升华。是的，这个过程我们也可以把它叫做冥想。而关于开源项目 [pure-admin](https://github.com/pure-admin) 及其生态的思想与源码解析的想法，就此迸发。

就这样在每个工作日的公交上，来来回回构思了无数次，要怎么去写这个系列。最后决定将文章的脉络分为两条线，交叉进行。

## 路线一：项目实践与解析

最早接触到 pure-admin 大约是在 2021 年 8 月，而同时在 9 月，`Vue3.0`正式发布为`Vue3.2`，也就是后来默认的 3.0 版本。[vue-pure-admin](https://github.com/pure-admin/vue-pure-admin)也在后续迅速的换上了 Composition-API，在当时 Vue3 兴起的风口下，vue-pure-admin 与我使用的技术栈`Vue3+Element-Plus+Vite+TS`完美的契合。期间 pure 持续保持着依赖版本的最新性与稳定性，我也几乎关注并保持更新了作者的每一个 commit，以至于后来作者在群里说我是研究他源码最深的人。一个在前端界家常便饭的话题，admin 后台管理系统，pure 作为个人开源项目 在 github 上从最早的小几百 star 到现在将近 8K 的 star，不得不说是非常成功的。当然，那时候我也针对某些细节对比了多个 star 更高的 admin 项目，比如后台管理系统的祖师爷[vue-element-admin](https://github.com/PanJiaChen/vue-element-admin)、[vue-vben-admin](https://github.com/anncwb/vue-vben-admin)，不管从技术新颖的程度上，还是初代 Vue3 开发者的代码编写的方式上来看，个人觉得 vue-pure-admin 对我来说是最合适的选择。

现在 pure 已经趋于稳定，也有了保姆级的文档[pure-admin-doc](https://yiming_chang.gitee.io/pure-admin-doc/)，里面详细介绍了项目的使用以及一些作者记录的开发问题。很庆幸，曾经作为 pure 团队的一员，参与了 pure-admin-doc 的搭建与初版编写（后来因为工作条件环境等原因无法继续编写）。但是对于 vue-pure-admin 的使用者来说，尤其是初级前端开发工程师和相关技术栈初次使用的开发者来说，依然存在很多的问题（尤其是怎么更改或者增加项目中某个模块的功能），更详细的文档与实践依然迫切需要。2022 年上半年，我写过一个系列的关于 pure 的实践文章（其实没写完），但是后来 pure 进行了大范围的重构，也就不适用了。恰逢在公司目前一个后台管理系统的发布之际，想趁这个时间点，重新梳理一下这个项目从 0-1 的实践，以及开发过程中，我们所遇到的问题做的选择，还有未来可能需要做的优化。比如我们怎么选择合适的插件依赖进行搭配，需不需要什么替代方案来渡过当前研发阶段。比如 Vue3 中，组件的开发，是使用 `defineComponent`、`setup`、`render` 函数还是 `tsx`。他们之间的通信，是选择 `pinia`、`defineProps/defineEmits`、`expose/ref`、`useAttrs`、`v-model`、`provide` / `inject` 还是 `mitt` 。又比如 `CSS` 处理器或框架的使用，`less` 还是 `sass`，`TailWind CSS`、`WindCSS` 还是 `Unocss`。

虽然说后台管理系统对于每个前端人都耳熟能详，也或许每个前端人都做过一个后台管理系统。但是想真正做好一个后台管理系统，其实并不简单。在 pure 的近千次 commit 中，在自己的项目落地实践中，发生的种种碰撞与摩擦，都值得去做一些反思与总结。将整个系统细分下来，都是前端这条路上的一个个缩影，每个缩影中，也都有着一个个前人走过的坑，它们可能不仅仅存在于后台，也可能是其他 PC 端软件、APP 亦或者小程序。不管是为了自己当前项目的总结复盘或技术的查漏补缺，还是围绕 pure 这个开源项目做详尽的开发文档，方便后来的优化与维护，都少不了这份文档的编写。

## 路线二：前端三年之约

时间非常的巧妙，在这个时间节点，我真正接触前端也将满三年整，再过四个月也将正式工作满三年。在最初步入这个行业的时候，我给自己定了 1-3-5 年的时间节点规划（具体我就不透漏咯）。毕业后，在一年结尾之时顺利地完成了跳跃，庆幸又觉得应该。将满三年之际，我仍在追逐自身的飞跃，坚定又相信自己 ~ 。

只有短短不到三年的工作经验，其实也只能说是一只前端界的小菜鸡（确实还是很菜~），所以更想在这个特殊的节点，对自己三年的一些沉淀做一轮总结。毕竟，你可以不断地更替你的技术，你身边可以走过一个又一个熟悉的人，但你一辈子，这样的三年只有一次。你需要在这个时候，发现一些过去的什么，预知以后的一些什么，这些不仅仅是技术，也不仅仅是什么人什么事，可能就只是你“自己”。然后，攒起对未来的，希望。

那么，就从写下这个系列文章开始。

写到这里，我改变了准备像曾经一样机械性写技术文章的想法，转向纪录片式口吻的方式进行这次实践与文档的编写，以我即你，你即我的角度去融入，去完成创作，以及对自己的一次洗礼。

有逻辑性地，想到什么写什么。这才是，我想要的技术，我想要的自己。

## 一些话

> 如果你刚入行前端，想要有一个详细可靠的从 0-1 的项目实践。

说实话现在的技术环境确实太缺乏这样的实践与分享了，一方面这样的分享太耗费时间与精力了，对于每个人来说都可以说是得不偿失，一方面类似的优异资源都需要付费（购买或者培训机构）。但并不是所有人都承担的起学费，并不是所有人都愿意去踩坑，也并不是所有人都能真正有所得。说这些其实是想到了自己大四那会的亲身经历，有很多人跟我一样是转行过来的，没有基础没有方向甚至可以说是没有对这行的认知，只知道计算机出路好。虽然当时我也是抱着对计算机的热爱走进这一行，但是面对高额的培训机构学费，面对未知又复杂的计算机环境，还是望而却步，选择自己在家苦苦自学。最后在考研失败、疫情爆发、基础薄弱的情况下，拖着疲惫的身躯进入艰难的就业市场，在已经毕业，七月底才找到第一份不太好的工作。所以，我想做一次这样的分享，因为我自己，曾经就是某某某，我，就是你们。

> 如果你也像我一样想对自己 3 年左右的工作进行一次技术和思想上的总结

3 年说长不长，说短也不短，若要用一句话来描述这三年的变化，我觉得这样能非常贴切的概括起来：当你准备开发一个模块的时候，从“我要怎么做”，到“我能不能这样做”，变成“我应该这样做”。比如 pure-admin，在拿到它的时候，我们通常会想，怎么将它使用到我们的实际项目中去。在使用的过程中，我能不能更改框架既定的某些逻辑，而不产生 Bug，保证项目不同环境的正常运行。同时在进行开发的时候，对框架某些地方提出质疑并给出优化方案（实际上在 pure 基本开发稳定前大家经常会在群里提出问题，当然你也可以提 issues 或者 pr）。这其实是一个比较常见也比较简单的过程，但我们不得不承认这是个人背后缓慢沉淀的结果。当然这也并不意味着你有了三年经验，会了这些，就是一名优秀的前端开发。真正优秀的人毕竟是少数，我们没有优异的学历，没有顶尖的天赋，但没有什么能阻止我们慢慢地努力，慢慢地沉淀。在接下来对项目的各个模块进行实践的时候，我也会基本遵循这个过程，去解决各种各样的问题（实际开发时，我们确实遇到了很多问题）。所以在这个系列里，跟着我一点一点地回忆，一起慢慢地记录总结。

> 如果你对自己未来的发展有一些疑虑（包括我自己）

第一年，你打好了基础，完成了跳跃，第三年，你有了自己的体系与思想，完成了飞跃，还有第五年在等着你，不是吗。关于未来发展这个问题，以及最近非常火的“前端已死”，已经有太多太多的讲述者，各方面不足的我其实是没有多少资格去讨论的，想说的就是，没有必要的焦虑，有必要的做好自己。

不过有几点，或许可以作为参考，其实也都是我自己在 3-5 年期间需要考量与定夺的事情（未来总是掌握在自己手中的）：

- 增强自己的保障性或不可替代性（技术方向上）：你可以选择这些技术业务——企业级中后台 node 业务（积累中成本高）、企业级 sass 服务（复杂）、基于 web 的重交互场景与业务（WebGL、WebGPU、地图、游戏、数据可视化等）、基于 web 的工业化软件（Web 版 excel、photoshop 等）
- 职业转型：这里基本就是往管理岗走了（当然也是我的主要考虑方向），走什么样的管理因人而异，同时还要看公司的发展看你的机遇
- 升学：有时候高点学历还是很有用的，其实对上面提到的管理岗就有不小的影响。大四没有上到自己想上的研究生其实是很遗憾的，虽然现实让我很难再去考一次，但是还是非常想能再有一次机会
- 考公：对某些人来说，可能是一个非常不错的选择。但是面对现在竞争的越来越激烈，以及国内某些形式的不容乐观，需谨慎。（我的话大概率是不会的，躺平不适合我，我的性格更倾向于坻坜前行）
- 热爱生活，身体健康：没有什么比你自己的身体与生活更重要，作为程序员的我们，请保证好自己的身体健康，请热爱属于你自己的生活！

这里就先说到这，接下来我们慢慢聊，希望这个系列的文章可能会给你带来一些帮助或者参考。另外，本人毕竟技术与经验有限，所写所述难免会有一些错误，欢迎大家评论留言一起讨论学习，请各位大佬们多多指教，一起进步！

## 开发准备

注： 本项目主要基于[vue-pure-admin](https://github.com/pure-admin/vue-pure-admin)源码，并参考其他优秀相关开源项目做一些优化，进行 0-1 的后台管理系统开发。

### 主要开发技术栈

- 核心前端框架：[Vue3.js](https://cn.vuejs.org/)（及其官方生态）
- UI 组件库：[Element-Plus](https://element-plus.gitee.io/zh-CN/)
- 项目构建工具：[Vite](https://cn.vitejs.dev/)
- 状态管理库： [Pinia](https://pinia.vuejs.org/zh/)
- Javascript 库：[Typescript](https://www.typescriptlang.org/zh/)

### 开发工具推荐

- 编辑器：`VSCode`
- 代码仓库：`Github`
- 调试及运行浏览器：`Google`（最新版谷歌浏览器）
- 聪明的 ChatGPT：懂的都懂，2023 年开发必备神器！
- AI Code：[Cursor](https://www.cursor.so/)。帮你写代码，帮你写注释，帮你解析源码，相信我，一定要用它！

- 流程图设计：`Draw.io`，交互、流程、架构
